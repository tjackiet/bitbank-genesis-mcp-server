現在のブランチに紐づく最新の Pull Request について、CodeRabbit のレビューコメントを確認し、対応が必要なものを修正してください。

## 手順

1. **PR の特定**: 現在の git ブランチ名を取得し、そのブランチの最新 PR を GitHub MCP ツール (`list_pull_requests` で `head` フィルタ) で探す
2. **レビューコメントの取得**: `pull_request_read` の `get_review_comments` でレビュースレッドを全件取得する
3. **CodeRabbit コメントのフィルタリング**: author が `coderabbitai` のコメントのみを対象とし、以下の重要度で分類する
   - **Critical** (`⛔ Critical`): 必ず修正
   - **Major** (`🔴 Major`): 必ず修正
   - **Minor** (`🟡 Minor`): 内容を確認し、妥当であれば修正する
   - それ以下 (Nitpick, Suggestion 等): スキップ（一覧として報告のみ）
4. **修正の実施**: Critical / Major / Minor (妥当なもの) について、コードを修正する
5. **コミット・プッシュ**: 修正をまとめてコミットし、現在のブランチにプッシュする
6. **結果の報告**: 以下をユーザーに報告する
   - 修正したコメントの一覧（重要度・ファイル・内容の要約）
   - スキップしたコメントの一覧（理由付き）
   - 対応不要だった場合はその旨

## 注意事項

- 既に resolved のスレッドはスキップする
- 修正内容が不明確、または大規模なリファクタが必要な場合はユーザーに確認する
- コミットメッセージは `fix: CodeRabbit レビュー指摘を反映` の形式で
